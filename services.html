Ideally, phylotastic components have web-service interfaces like these, which you can access with your browser (XML- or JSON-formatted output may be invisible until you choose "view source").

  • DateLife example

    Clicking the link above executes the URL http://datelife.org/cgi-bin/R/result?taxa=Rhinoceros_unicornis%2CEquus_caballus%2CMus_musculus&format=newickmed&partial=liberal&useembargoed=no&uncertainty=100, which gives the result ((Rhinoceros_unicornis:55.8,Equus_caballus:55.8):40.3,Mus_musculus:96.1);, indicating that Rhinoceros and Horse diverged 55.8 MYA.

  • Pruner (MapReduce)

    This service invokes a pruner using an URL API as follows: phylotastic.cgi?format=newick&tree=mammals&species=Homo_sapiens,Pan_troglodytes,Gorilla_gorilla where

    • format is one of { Newick, NeXML, NEXUS, PhyloXML }
    • tree is one of { mammals, fishes, angiosperms, phylomatic, tol }
    • species is a comma-separated list of taxonomic identifiers

  • TNRS meta-service

    The TNRS meta-service accepts a list of names, and returns a JSON document (in progress: DwC-A) that, for each name, lists the matches from all of the back-end TNRSs that it aggregates. A hypothetical intelligent client would apply rules to decide which matches to use.

  • Phylomatic URL API

    Phylomatic has a new web service interface (click the link above for explanation) that accepts URLs like this: http://phylodiversity.net/phylomatic/pm/phylomatic.cgi?format=xml&tree=annonaceae%2Fannona%2FAnnona_cherimola%0D%0Aannonaceae%2Fannona%2FAnnona_muricata.

  • TreeStore

    The page above provides links to several queries that will retrieve sample trees from a treestore. The server uses web2py to provide access to trees stored in a Virtuoso triplestore.
